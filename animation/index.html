Chat App animations

Note that my mouse is not visible in screen recordings below. I think this is an open issue with Ubuntu 22.04's built-in screen recorder.

1. Darken and expand clickable box on hover

When hovered over, the chat box darkens and the chat name expands, both changes occurring quickly but smoothly (a subtle animation, but slightly less jarring than instantaneous change).

2. Modal fade-in/out

The settings popup modal fades in and out.

3. Smooth scrolling

When the user sends a message (or receives a message while resting at the bottom of the chat window), the chat window scrolls smoothly to show the new message. Note: to my initial confusion and subsequent dismay, the behavior: smooth option passed to scrollIntoView() works in Firefox but has known bugs in Chrome (and it actually sabotages the autoscroll functionality in my app entirely). I did not figure out how to achieve the effect otherwise, so the code in my final submission lacks this animation.

4. Incognito banner moves in/out

As the user toggles Incognito mode, the banner moves in or out accordingly.

5. Tooltip moves in/out

When the user searches for a non-existent username or requests a new username, the corresponding feedback tooltip appears to move up into view, stays for a few seconds, and then moves back down out of view.
